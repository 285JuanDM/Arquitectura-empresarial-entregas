## 🧠 Actividades realizadas en clase

Durante la sesión se analizó el caso base **GobData**, un portal estatal para la gestión de trámites ciudadanos que maneja información personal y datos sensibles como identificación, historial médico y certificados digitales.

El equipo identificó como flujo crítico el proceso de **registro, autenticación y gestión de trámites por parte del ciudadano**, debido a que involucra la recolección, almacenamiento y uso de datos personales bajo un contexto regulado.

A partir de esto, se aplicó un **checklist de cumplimiento normativo**, basado en principios de la **Ley 1581 de 2012 (Habeas Data)** y controles de **ISO/IEC 27001**, evaluando aspectos como:

* Consentimiento del usuario
* Finalidad del tratamiento de datos
* Control de accesos
* Seguridad de la información
* Trazabilidad y auditoría
* Retención de datos

El análisis se estructuró en una tabla donde se evaluó el nivel de cumplimiento (cumple, parcial o no cumple), acompañado de justificaciones basadas en el tipo de información gestionada por el sistema.

Durante la clase se logró identificar posibles brechas, especialmente en escenarios donde no se garantiza el consentimiento explícito del usuario o donde podrían existir riesgos de acceso no autorizado a datos sensibles.
