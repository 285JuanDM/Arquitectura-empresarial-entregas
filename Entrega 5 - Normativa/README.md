# 📖 Descripción del Proyecto

Este repositorio contiene el desarrollo del taller de **evaluación de cumplimiento normativo y protección de datos** en el curso de Arquitectura Empresarial.

El objetivo principal fue analizar el nivel de cumplimiento de sistemas tecnológicos frente a normativas como la **Ley 1581 de 2012 (Habeas Data)** y estándares internacionales como **ISO/IEC 27001**, mediante la aplicación de listas de control (checklists).

El trabajo se dividió en dos casos principales:

* **Caso base – GobData:** se realizó un análisis de cumplimiento sobre un portal gubernamental de trámites ciudadanos, evaluando aspectos como consentimiento, manejo de datos sensibles, control de accesos y trazabilidad.
* **Caso real – TransCapital S.A.S:** se aplicó el checklist al proceso de **generación de FUEC**, identificando brechas relacionadas con el uso de archivos Excel, ausencia de controles de seguridad, falta de políticas de privacidad y riesgos en el manejo de datos personales.

Para el desarrollo del análisis se utilizó **Excel** para la construcción de tablas de cumplimiento y **Markdown** para la documentación, permitiendo estructurar claramente los criterios evaluados, el nivel de cumplimiento y las recomendaciones.

# 🎯 Objetivo

Aplicar un enfoque de **evaluación de cumplimiento normativo** para identificar brechas en la gestión de datos personales y seguridad de la información en sistemas tecnológicos.

A través del análisis se buscó identificar:

* Nivel de cumplimiento frente a normativas de protección de datos
* Riesgos legales asociados al manejo de información personal
* Debilidades en controles de seguridad (acceso, trazabilidad, almacenamiento)
* Brechas en políticas organizacionales y procesos
* Oportunidades de mejora alineadas con buenas prácticas

Este enfoque permite garantizar la protección de la información, reducir riesgos legales y mejorar la confianza en los sistemas analizados.

# ↗️ Análisis de Cumplimiento Normativo

## Caso Base – 🏛️ GobData (Portal de Trámites Ciudadanos)

El análisis aplicado a **GobData** permitió evaluar el cumplimiento de un sistema que gestiona grandes volúmenes de datos personales y sensibles.

Se identificaron aspectos clave como la necesidad de garantizar el consentimiento informado del usuario, implementar controles de acceso robustos y asegurar la trazabilidad de las operaciones. También se evidenció la importancia de contar con políticas claras de retención de datos y protección de información sensible.

El modelo de checklist permitió estructurar la evaluación en diferentes categorías, facilitando la identificación de brechas normativas y riesgos asociados al manejo de información en entornos gubernamentales.

## Caso Real – 🚌 TransCapital S.A.S

El análisis de cumplimiento en **TransCapital S.A.S** se enfocó en el proceso de **generación de FUEC**, el cual actualmente se realiza mediante archivos Excel y procesos manuales.

El checklist evidenció múltiples incumplimientos normativos, especialmente en aspectos como:

* Falta de consentimiento para el uso de datos personales
* Ausencia de controles de acceso y roles definidos
* Inexistencia de trazabilidad y auditoría
* Falta de políticas de privacidad y retención de datos
* Riesgo de exposición de información sensible

Estos hallazgos reflejan un bajo nivel de madurez en la gestión de la información, lo que representa riesgos legales y operativos significativos. A partir del análisis, se propusieron mejoras orientadas a la implementación de controles de seguridad, formalización de políticas y adopción de buenas prácticas.
